## [S1] 一句话描述

把当前 1v1 单回合 PK 升级为 **4v4 编队接力战**——双方各上 4 个老婆，速度先手、回合制、死亡换人、战败结算，整个战斗过程通过主动消息流（多回合贴文 + 结算贴）完整呈现。

## [S2] 核心变化 vs 当前 PK

| 维度 | 当前（Phase 4） | 升级后 |
|------|----------------|--------|
| 战斗形式 | 单回合战报 | 多回合接龙 |
| 阵容 | 1v1（主老婆或指定编号） | 4v4 编队 |
| 战斗过程 | 1 条消息定胜负 | 最多 12 回合，0.8-1.2s 间隔多条消息 |
| 先手规则 | 攻方先手 | 速度判定（赛尔号式） |
| 老婆特性 | 纯数值 | S2 简化被动（按稀有度派生） |
| 战斗状态 | 无持久化 | `pk_battles.json` 持久化 |
| 锁定兼容 | 与 PK 无关 | 锁定老婆可进编队但 PK 战力 -15% |
| 战斗超时 | 不可能 | 12 回合上限 + 按剩余 HP 判 |
| 战斗取消 | 不可 | 管理员可 `老婆 战斗取消` |

### [S3.4] 元素派生（沿用 Phase 4 决策）

```python
def derive_element(meta: WifeMeta) -> str:
    """按 base_stats 最高项派生（不持久化 WifeMeta.element）
    
    锁定规则：
      atk 最高 → 力量（克制敏捷）
      hp 最高 → 敏捷（被力量克制）
      def 最高 → 智力（克制力量）
      并列按 atk > hp > def 优先级
    """
    a, d, h = meta.base_stats["atk"], meta.base_stats["def"], meta.base_stats["hp"]
    if a >= d and a >= h: return "力量"
    if h >= a and h >= d: return "敏捷"
    return "智力"
```

### [S3.5] 双层被动：稀有度 × 元素

每个老婆同时持有 2 个独立被动，二者按乘法叠加，形成 **4 × 3 = 12 种角色差异**。

### [S3.6] 元素克制倍率细化

克制循环：**力量 → 敏捷 → 智力 → 力量**

| 关系 | 倍率 |
|------|------|
| 完全克制（克到被克方） | **×1.30** |
| 被克制 | **×0.75** |
| 同元素 | ×1.00 |

> 从 Phase 4 的 1.20/0.80 加深到 1.30/0.75，**让克制关系更有战术权重**。
> 同一个完美克制组合的胜率提升更明显，反之被克制方更难翻盘。

### [S3.7] 战斗状态层（战斗中累积 · 让战斗有"温度"）

每个老婆持有最多 4 种状态层，**战斗回合动态累积**，影响下一回合表现。

```python
@dataclass
class BattleStatusLayer:
    qi_po: int = 0      # 气魄（攻击命中后 +1，最多 5 层，每层 +3% atk）
    weak_point: int = 0 # 弱点（被攻击命中后 +1，最多 3 层，每层 +5% 受伤）
    bloodlust: int = 0  # 血性（5 回合后激活 +10% atk，8 回合后 +15% atk，叠加）
    frenzy: bool = False  # 狂暴（当前 HP < 30% 时激活，atk +25% / speed -25%）
```

| 状态 | 触发条件 | 累积效果 | 策略意义 |
|------|---------|---------|---------|
| **气魄** | 攻击命中对方 | 每层 atk +3%，最多 5 层（+15%） | 强者越强，鼓励持续输出 |
| **弱点** | 被攻击命中 | 每层受伤 +5%，最多 3 层（+15%） | 被压制方会越来越脆 |
| **血性** | 5/8 回合自动激活 | 5T: +10% atk / 8T: +15% atk | 第 4 个老婆"米卡莎"可能后期翻盘 |
| **狂暴** | 当前 HP < 30% | atk +25% / speed -25% | 残血翻盘核心机制，鼓励拖延 |

#### 状态层战术影响举例

```
回合 3 时双方状态：
  攻方三笠: 气魄=2 (atk+6%) + 血性尚未激活 + HP 380/500
  守方蝴蝶忍: 弱点=2 (受击+10%) + 血性尚未激活 + HP 200/450
  
回合 6 时：
  攻方三笠: 气魄=5 (上限, atk+15%) + 血性Lv1 (atk+10%) = 总 atk +25%
  守方蝴蝶忍: 弱点=3 (上限, 受击+15%) + HP 90/450（狂暴激活）
  攻方三笠 的伤害已经翻了 25%，对方被击伤害还多 15% —— 一波带走
```

```
回合 9 时（米卡莎刚上）：
  血性Lv2 (atk+15%) + 狂暴 HP 88/200 = atk +40%，speed -25%
  ── 但此时攻击翻倍反而变成被先手的目标
  ── 残余最后一击有爆炸力但需要队友铺垫
```

> **状态层让战斗从"看数字胜负"变为"看状态演化"**——长期 PK 经验玩家能看出"对方弱点 3 层，强攻一波"这种战术判断。

## [S4] 命令设计

### [S4.1] `老婆 编队`（新增）

| 子命令 | 行为 | 例 |
|--------|------|-----|
| `老婆 编队` | 查看当前编队 | "你还没有设置编队。使用 `老婆 编队 <编号1> <编号2> <编号3> <编号4>` 一次性设置 4 位老婆。" |
| `老婆 编队 1 3 2 4` | 按用户指定顺序设置 4 位 | 1 号位放 wid_a，3 号位放 wid_b... |
| `老婆 编队 默认` | 恢复默认 [1,2,3,4]（按持有列表） | 当持有老婆 >= 4 时有效 |
| `老婆 编队 清除` | 清空当前编队（下一次 PK 用默认并提示） | |

**校验**：
- 编号必须是自己持有的老婆
- 必须在持有列表去重后的前 N 个里（不能重复同一个 wid）
- 长度 4 最佳，但允许 1~3 长度
  - 长度 < 4：PK 提示"当前编队 N 位，不足 4 位，战斗将以 Nv4 进行"——玩家可选择继续 / `老婆 编队`

### [S4.2] `老婆 PK @群友`（重写）

启动条件：
1. 攻击者编队至少 1 位（不足 4 提示）
2. 防御者编队至少 1 位（不足 4 时：**用防御者默认编队 [1..N] 并群播报"@xxx 还没设置编队，系统临时用默认编队：xxx"**）
3. 双方 24h 内未 PK 过彼此
4. 双方至少有 1 个老婆（持有列表非空）

启动后：
- 创建 `PkBattle`，写入 `pk_battles.json`
- 立刻发送 **启动贴**（双方编队 + 战力对比）
- `PkService._auto_advance_battle(battle)` 协程开始推回合

### [S4.3] `老婆 战斗状态`（新增，只对自己）

返回当前群正在进行的战斗摘要（如果该用户在战斗中）：
```
你在 [gid] 群参与一场 4v4 接力战！
  你的编队 [攻击方/防御方]
  当前回合 N / 12
  你的存活: N/4
  
还有 X 条回合消息待推送。
```

### [S4.4] `老婆 战斗取消 <battle_id>`（管理员命令，ABK 模式）

`adb_cancle_battle(gid, battle_id)` 立即结束战斗，按剩余 HP 比例结算（无输赢，无奖励），所有数据回滚或保留（见 §S7.5）。

### [S4.5] 兼容与回退

**旧 `老婆 PK @群友 [我方编号] [对方编号]` 语法保留**：
- 检测到带编号参数 → 走旧逻辑（1v1 单回合）
- 不带编号 → 走 4v4 接力战（新）

**回退策略**：如果 `pk_battles.json` 异常或战斗被禁用，`老婆 PK @群友` 退回 1v1 单回合。

## [S5] 战斗状态机

```
                PK 启动
                   ↓
              [INIT]
              启动贴
            ↓ 主动消息
           [TURN]
        每回合：先手判定 → 双方攻击 → 死亡换人 → 回合贴
            ↓ 循环 ×12
      ┌─ 一方全死 → [SETTLE]（结算贴）
      └─ 12 回合到 → [SETTLE]（按剩余 HP 判）
                   ↓
              [SETTLE] 结算贴
                   ↓
                [DONE]
            → 清理 pk_battles.json
```

**主动消息推送**：
- 由 `_auto_advance_battle` 协程负责
- 每回合间 sleep 0.8s（如果有人设定可配置 `pk_turn_delay_ms`）
- 启动贴 / 结算贴 是独立消息，间隔 1.5s

## [S6] 单回合算法

### [S6.1] 速度判定（双层被动 + 状态层）

```python
def calc_speed(member: FormationMember, status: BattleStatusLayer) -> int:
    """本回合在场老婆的速度，考虑双层被动 + 状态层。
    
    来源叠加顺序：
      1) 基础速度 = atk × 0.3 + hp × 0.2
      2) R 卡被动：speed ×1.10
      3) 敏捷元素被动：speed ×1.10
      4) 锁定老婆：speed ×0.85（用户决策 L1）
      5) 狂暴状态（HP < 30%）：speed ×0.75
    """
    spd = int(member.base_atk * 0.3 + member.base_hp * 0.2)
    if member.rarity == "R":
        spd = int(spd * 1.10)        # 稀有度被动：速度 +10%
    if member.element == "敏捷":
        spd = int(spd * 1.10)        # 元素被动：速度 +10%
    if member.is_locked:
        spd = int(spd * 0.85)        # L1 锁定惩罚
    if status.frenzy:
        spd = int(spd * 0.75)        # 狂暴扣速
    return spd

def pick_first_striker(atk: FormationMember, df: FormationMember,
                      atk_status: BattleStatusLayer, df_status: BattleStatusLayer) -> str:
    sa = calc_speed(atk, atk_status)
    sd = calc_speed(df, df_status)
    if sa > sd: return "atk"
    if sd > sa: return "def"
    # 平速度：随机（用 rng_state 保公平 + 可复现）
    return "atk" if rng.random() < 0.5 else "def"
```

### [S6.2] 伤害公式（双层被动 + 状态层 · 完整版）

```python
def calc_damage(attacker: FormationMember, defender: FormationMember,
                atk_status: BattleStatusLayer, df_status: BattleStatusLayer,
                turn_idx: int, rng) -> dict:
    """完整伤害公式，融合：
       - 双层被动（稀有度 + 元素）
       - 元素克制（深化 1.30 / 0.75）
       - 4 状态层（气魄/弱点/血性/狂暴）
       - 命中 90% / 暴击 10% / 浮动 ±20%
       - 打工惩罚 / 亲密加成 / 锁定惩罚 / SR 减伤
    """
    # === 1. 命中 90% ===
    if rng.random() > 0.90:
        return {"hit": False, "dmg": 0, "crit": False, "combo": False}
    
    # === 2. 暴击率（基础 10% + 智力元素被动 +5% + SSR 稀有度 +10%）===
    crit_rate = 0.10
    if attacker.rarity == "SSR":
        crit_rate += 0.10  # 稀有度被动：暴击 +10%
    if attacker.element == "智力":
        crit_rate += 0.05  # 元素被动：暴击 +5%
    is_crit = rng.random() < crit_rate
    
    # === 3. 暴击倍率（智力受击 -15%）===
    if is_crit:
        crit_mult = 1.5
        if defender.element == "智力":
            crit_mult *= 0.85  # 元素被动：受暴减伤
    else:
        crit_mult = 1.0
    
    # === 4. 元素克制倍率（深化 1.30 / 0.75）===
    element_mult = {
        ("力量", "敏捷"): 1.30, ("敏捷", "力量"): 0.75,
        ("敏捷", "智力"): 1.30, ("智力", "敏捷"): 0.75,
        ("智力", "力量"): 1.30, ("力量", "智力"): 0.75,
    }.get((attacker.element, defender.element), 1.0)
    
    # === 5. 攻击方 atk 倍率（力量元素被动 ×1.15 + 狂暴 +25%）===
    atk_mult = 1.0
    if attacker.element == "力量":
        atk_mult *= 1.15  # 元素被动：力量 atk +15%
    atk_bonus_total = (atk_mult - 1.0) + (0.25 if atk_status.frenzy else 0.0)
    
    # === 6. 状态层（气魄 + 血性，作用于攻击方 atk）===
    qi_po_bonus = atk_status.qi_po * 0.03  # 每层 +3%，最多 5 层（+15%）
    bloodlust_bonus = 0.0
    if turn_idx >= 8:
        bloodlust_bonus = 0.15    # 血性Lv2：atk +15%
    elif turn_idx >= 5:
        bloodlust_bonus = 0.10    # 血性Lv1：atk +10%
    
    atk_layer_mult = 1.0 + qi_po_bonus + bloodlust_bonus
    
    # === 7. 防御方弱点层（受击 +5%/层，最多 3 层 +15%）===
    weak_point_mult = 1.0 + df_status.weak_point * 0.05
    
    # === 8. 打工惩罚（沿用 Phase 4）===
    work_mult = {
        "normal":    0.85,
        "overtime":  0.75,
        "expedition": 0.65,
    }.get(attacker.work_mode, 1.0)
    
    # === 9. 亲密加成（沿用 Phase 4）===
    intimacy_mult = 1 + attacker.intimacy / 500.0
    
    # === 10. 锁定惩罚（L1 决策）===
    lock_atk_mult = 0.85 if attacker.is_locked else 1.0
    
    # === 11. SR 减伤（被 SSR/SR 攻击时）===
    if defender.rarity == "SR" and attacker.rarity in ("SSR", "SR"):
        damage_taken_mult = 0.90
    else:
        damage_taken_mult = 1.0
    
    # === 12. 伤害合成 ===
    atk_base = attacker.base_atk * (1.0 + atk_bonus_total) + attacker.base_def * 0.3
    raw = atk_base \
        * atk_layer_mult \
        * crit_mult \
        * element_mult \
        * work_mult \
        * intimacy_mult \
        * lock_atk_mult \
        * damage_taken_mult \
        * weak_point_mult
    
    # === 13. 浮动 ±20%（你指定的）===
    jitter = 0.8 + rng.random() * 0.4
    dmg = max(1, int(raw * jitter))
    
    return {"hit": True, "dmg": dmg, "crit": is_crit, "combo": False}
```

#### 伤害公式乘数一览（实施参考）

### [S6.3] 死亡换人

每只老婆死亡后立刻从编队位置+1 处替补，没有则该方败北。
- 阵亡时：查找编队内下一存活者，标记为 `is_active`
- 双方同时阵亡：各换下一个
- 一方无下一个：该方失败

### [S6.4] 单回合最终流程（伪代码）

```python
async def do_turn(battle: PkBattle, round_no: int):
    atk_member = battle.atk_formation[battle.active_idx[0]-1]
    df_member  = battle.def_formation[battle.active_idx[1]-1]
    atk_status = battle.atk_status_layers[battle.active_idx[0]-1]
    df_status  = battle.def_status_layers[battle.active_idx[1]-1]
    
    # 1. 速度判定（双层被动 + 状态层）
    first = pick_first_striker(atk_member, df_member, atk_status, df_status)
    
    # 2. 先手攻击 + 累积状态层
    await attack(first_one, second_one, battle, round_no, "first", atk_status, df_status)
    
    # 3. 后手反击（如果还活着）
    if second_one.is_alive:
        await attack(second_one, first_one, battle, round_no, "retaliate", df_status, atk_status)
    
    # 4. 死亡换人
    swap_if_dead(battle, "atk")
    swap_if_dead(battle, "def")
    
    # 5. 累积回合状态（血性 / 狂暴判定）
    accumulate_status_layers(battle, round_no)
    
    # 6. 检查胜负
    return check_end(battle)
```

### [S6.5] 状态层计算与触发

```python
def accumulate_status_layers(battle: PkBattle, turn_idx: int) -> None:
    """每回合结束 + 每次攻击结算时调用，更新双方当前在场老婆的状态层。"""
    
    for side in ("atk", "def"):
        layer = battle.{f"{side}_status_layers"}[battle.active_idx[side_idx]-1]
        member = battle.{f"{side}_formation"}[battle.active_idx[side_idx]-1]
        
        # 1. 气魄 / 弱点：在 attack() 里实时增减
        
        # 2. 血性：5/8 回合自动激活（持久战奖励）
        if turn_idx >= 8:
            layer.bloodlust = 2   # +15% atk
        elif turn_idx >= 5:
            layer.bloodlust = 1   # +10% atk
        
        # 3. 狂暴：当前 HP < 30% 时激活
        hp_ratio = member.current_hp / member.base_hp
        if hp_ratio < 0.30:
            layer.frenzy = True

def on_hit_apply_layers(attacker_status, defender_status) -> None:
    """每次攻击命中后调用：攻方+1 气魄、守方+1 弱点。"""
    if attacker_status.qi_po < 5:
        attacker_status.qi_po += 1
    if defender_status.weak_point < 3:
        defender_status.weak_point += 1
```

## [S7] 边界与异常

### [S7.1] 战斗超时
12 回合到达后仍未分胜负：按双方总剩余 HP 比（`atk_total_hp / def_total_hp`）判胜。

### [S7.2] 战斗被外部事件打断
- NTR 复仇：`PkBattle` 期间，被牛方的老婆属于战斗中（`is_alive` 状态），外部 NTR 不影响
- 打工结算：战斗中老婆的打工时间到了，PK 优先，PK 结束后再结算打工
- 离婚：战斗中老婆不能离婚，命令返回"她正在战斗，结束后再说"

### [S7.3] 同对手 24h 防刷
沿用 Phase 4 的 `pk_pairs.json`（`sort(uid1|uid2): ts`）。战斗启动前检查。

### [S7.4] 编队合法性校验
- 编队内老婆必须自己持有
- 锁定老婆可进编队但有战力惩罚
- 编队变更不影响正在进行的战斗（战斗中冻结编队）

### [S7.5] 战斗取消（管理员）
`老婆 战斗取消 <battle_id>` 立即终止，**所有币/积分/亲密度变化回滚**（在战斗中没有持久化任何变化，无须回滚）。

### [S7.6] bot 重启与战斗丢失
- 启动时扫描 `pk_battles.json`：发现 status=active 的，立即按 end_reason=admin_cancel 结算，发送"由于系统维护，战斗中断"
- 这是兜底逻辑，没有"断点续战"——战斗一定从头开始，避免跨重启的复杂度
