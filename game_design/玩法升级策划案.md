# 《二次元老婆》玩法升级策划案（减负版 v2 · 迭代深化）

> 定位：QQ 群聊机器人小游戏（AstrBot 插件 `astrbot_plugin_animewifexI`）
> 核心支柱：**社交 + 竞争**为主，**成就 + 探索**为辅
> 原则：每个系统都要轻量、可一句话说清、能融入核心循环
>
> **v2 迭代说明**：本版本在 v1 减负版基础上做了深化。核心改动：①删除防连环牛，鼓励群聊动态平衡；②新增老婆打工系统（风险-收益博弈）；③NTR 复仇链路完整化；④PK 数值验证与防刷；⑤经济压力测试；⑥新手保护；⑦实施风险控制。

## 〇、设计哲学

### 四条不可妥协的原则

1. **群聊小游戏 ≠ 手游**。所有系统必须能在群聊对话流中完成，命令式交互是唯一入口。
2. **负反馈必须有上限**。玩家单日最大负体验 = 被牛 1 次 + PK 连败 3 次。超过即触发保护机制。
3. **产出必须 ≥ 投入感知**。玩家"感觉"自己每天都在变强，签到连续奖励、PK 段位晋升、打工收益，都是这种"可见进步"的载体。
4. **🆕 风险与收益必须成正比**。想赚钱就要承担被牛风险（打工系统）；想安全就要放弃收益。这是动态平衡的核心。

### 玩家画像与动机匹配

| 玩家类型 | 占比预估 | 核心动机 | 本案对应系统 |
|---------|---------|---------|------------|
| 休闲收集型 | 50% | 抽老婆、看图、填图鉴 | 抽卡、图鉴里程碑、亲密度等级 |
| 社交互动型 | 25% | 和群友玩、送东西、排名 | 赠送道具、排行榜、PK 段位 |
| 竞争好胜型 | 20% | 牛别人老婆、PK 上分 | NTR、PK 段位、复仇令牌、🆕 打工盯梢 |
| 深度养成型 | 5% | 把老婆养到满亲密度、凑羁绊 | 亲密度 Lv.5、羁绊加成、约会 |

## 一、现状核心问题（4 条）

1. **循环断裂**：抽完老婆后无事可做，养成→竞争→奖励没有连成闭环
2. **NTR 惩罚过重**：亲密度归零（`ownership_service.py:715`），玩家投入瞬间清零 → 退坑
3. **PK 零策略**：纯比数值 + ±20% 随机，败方零收益，没人愿意打
4. **新手真空期**：新玩家初始 50 币，抽完 1 次免费老婆后无事可做，前 3 天是流失高危期

## 二、核心循环（必须连起来）

### 2.1 当前问题

```
签到 → 抽老婆 → 摸头 → 等明天
         ↑                ↓
      （断点）          （无事可做）
```

### 2.2 新循环：一条闭合的链

```
签到领币 → 抽老婆(消耗券) → 养成亲密度(消耗币)
    ↑                                ↓
  产出币                        亲密度提升战力
    ↑                                ↓
商城买券 ← 消耗币 ← 排行/任务 ← PK/NTR(竞争)
    ↑                                ↓
  🆕打工收入 ← 🆕老婆打工(风险) ← 亲密度够高才有底气
```

**关键设计**：
- **PK/NTR/打工 是循环的三台发动机**：竞争行为产出币和排名；打工提供高风险高收益选择
- **打工让"闲着"变成"选择"**：老婆不打工就浪费时间，打工就有被牛风险——这是动态博弈
- **排行榜把竞争结果变成社交展示**：打工收入榜、NTR 猎杀榜、PK 段位榜

### 2.3 循环验证（玩家一天的行为）

| 时间 | 行为 | 花币 | 赚币 | 风险 | 竞争/社交 |
|------|------|------|------|------|-----------|
| 早上 | 签到 | — | +15~45 | — | — |
| 早上 | 抽老婆（免费/券） | 0~30 | — | — | — |
| 中午 | 摸头 x2 + 对话 | -10 | +5 | — | — |
| 下午 | 🆕 让老婆打工 | -10(启动) | +15~25(完成) | ⚠️ 被牛概率 ×1.5 | — |
| 下午 | PK @某人 | — | +5~15 | — | ✅ 竞争 |
| 晚上 | NTR @某人 | — | 0~补偿 | — | ✅ 竞争 |
| 晚上 | 任务领奖 | — | +10 | — | — |
| 晚上 | 商城买券 | -30~270 | — | — | — |

**产出 ≈ 50~100/天（含打工），消耗 ≈ 40~310/天**，经济更活跃，玩家有更多选择。

## 四、NTR 体验重构

### 4.1 问题

当前：NTR 成功 → 亲密度归零。玩家投入数周的养成瞬间清零，严重违反"不过度惩罚"原则。

### 4.2 新方案：降级 + 补偿 + 令牌

```
NTR 成功后（被牛方）：
  ① 亲密度保留 50%：intimacy = original // 2（不归零）
  ② 老婆币补偿 = 失去的亲密度 × 2（封顶 50 币）
  ③ 复仇令牌 x1（必给，用于复仇时成功率额外 +20%）
  ④ 悲伤 buff：24h 内 PK 战力 +20%（仅对 NTR 攻击者生效）
  ⑤ 新手保护：注册当天或总老婆数 < 3 的玩家，被牛后亲密度保留 75%
```

```
NTR 成功后（攻击方）：
  ① 老婆转移（不变）
  ② 亲密度从 0 开始（不变）
  ③ 连续 NTR 衰减：对同一目标连续成功，每次成功率 ×0.7
  ④ 作恶值：每次 NTR 成功 +1，累积到 5 时全群提示"恶名昭著"
  ⑤ 🆕 打工老婆额外奖励：如果牛的是打工中的老婆，打工收入归攻击者
```

### 4.3 NTR 成功率动态公式

```
final_prob = base_prob(0.15)
  × (0.5  if 目标亲密度 >= 80)     # 亲密度护盾
  × (0.3  if 目标有保护符)          # 保护符
  × (0.0  if 目标已锁定)            # 锁定卡
  × (1.8  if 是复仇)               # 复仇加成
  × (0.7  if 攻击者对同一目标连续成功)  # 防霸凌
  × (0.5  if 目标是新手)            # 新手保护
  × (1.5~2.5 if 目标正在打工)       # 🆕 打工期间防御降低
```

**关键设计**：
- **不设防连环牛**：群聊环境下，多人围攻大佬是健康的动态平衡机制
- 打工老婆防御降低是主动选择的结果（想赚钱就要承担风险）
- 连续衰减只对同一目标生效（防止同一个人被反复牛）

### 4.4 复仇令牌完整链路

```
被牛 → 获得 revenge_token x1（存入 inventory）
         ↓
复仇命令（老婆 复仇 @攻击者）
         ↓
检查令牌 + 复仇窗口（24h）
         ↓
有令牌：消耗令牌，成功率 = base × 1.8 × 1.2 = base × 2.16
无令牌：成功率 = base × 1.8（仍可复仇，只是没额外加成）
         ↓
复仇成功：夺回老婆，亲密度恢复到被牛前的 75%
复仇失败：令牌不返还，但获得 10 币安慰奖
         ↓
无论成败，last_ntr_by 清空，复仇链结束
```

### 4.5 作恶值系统

- 每次 NTR 成功，攻击者 `evil_points += 1`
- 达到 3：查询老婆时显示"⚠️ 此人已牛过 3 次"
- 达到 5：全群播报"恶名昭著"，被牛者补偿翻倍
- 作恶值每月清零

## 五、🆕 老婆打工系统（核心新增）

### 5.1 设计理念

**打工 = 主动选择风险换取收益**

- 打工时老婆不在身边 → 更容易被牛走
- 打工成功 → 赚钱 + 亲密度小加成
- 打工期间被牛 → 老婆转移 + 打工收入归攻击者

这是一个**正和博弈**：
- 玩家赚到了币（打工收益）
- 群聊产生了动态（有人盯梢打工老婆）
- 大佬面临困境：不打工缺钱，打工有风险
- 小号有机会：盯紧打工中的大佬老婆

### 5.2 打工模式

| 模式 | 耗时 | 启动费 | 收益 | NTR 概率加成 | PK 战力 | 亲密度加成 |
|------|------|--------|------|-------------|---------|-----------|
| 普通打工 | 1h | 10 币 | 15~20 币 | ×1.5 | -15% | +2 |
| 加班打工 | 2h | 20 币 | 28~35 币 | ×2.0 | -25% | +5 |
| 远征打工 | 3h | 30 币 | 50~60 币 | ×2.5 | -35% | +8 |

**命令**：
```
老婆 打工           ← 普通打工（默认）
老婆 打工 加班       ← 加班打工
老婆 打工 远征       ← 远征打工
```

### 5.3 打工流程

```
玩家发送命令
    ↓
检查条件：有主老婆 + 余额 ≥ 启动费
    ↓
扣除启动费，老婆进入"打工中"状态
    ↓
【1/2/3 小时等待期】
    ↓
此时老婆可被 NTR（概率提升 1.5x~2.5x）
此时 PK 战力 -15%~-35%
    ↓
等待期结束，检查是否被牛
    ↓
┌─ 未被牛：发放收益 + 亲密度加成 + 记录连续天数
│
└─ 被牛：老婆转移给攻击者 + 打工收入归攻击者（额外奖励）
```

### 5.4 打工状态数据结构

`UserProfile` 新增：
```python
work_status: Optional[Dict[str, Any]] = None
# None = 没在打工
# {"mode": "normal", "start_ts": 1234567890, "end_ts": 1234571490, "streak": 3}
```

`Ownership` 新增：
```python
is_working: bool = False
work_mode: str = ""  # "normal" / "overtime" / "expedition"
work_ends_at: int = 0  # 结束时间戳
```

### 5.5 打工期间的 NTR 交互

当有人尝试 NTR 打工中的老婆时：

```python
# NTR 概率计算（在 try_ntr 中）
final_prob = base_prob
  × ...（其他因素）...
  × (1.5 if normal打工 / 2.0 if overtime / 2.5 if expedition)
```

**设计意图**：
- 打工老婆更容易被牛 = 打工有代价
- 但不是必被牛，只是概率提升
- 大佬可以选择让低亲密度老婆打工（被牛也无所谓）
- 或者选择高亲密度老婆打工（风险高但收益也高）

### 5.6 打工期间的 PK 交互

当打工中的老婆出战 PK 时：

```python
# PK 战力计算
if ownership.is_working:
    work_penalty = {"normal": 0.15, "overtime": 0.25, "expedition": 0.35}
    final_power = base_power * (1 - work_penalty[work_mode])
```

**设计意图**：
- 打工中的老婆不在状态，PK 战力降低
- 这是另一个打工代价
- 玩家必须权衡：是让老婆打工赚钱，还是留着打 PK？

### 5.7 🆕 打工升级玩法

#### a) 连续打工奖励

| 连续天数 | 额外奖励 |
|---------|---------|
| 3 天 | +5 币 |
| 5 天 | +10 币 + 单抽券 x1 |
| 7 天 | +20 币 + 称号「打工之王」 |

连续天数在被牛或断签时重置。

#### b) 打工合约（提前锁定收益）

```
老婆 打工 合约 [模式]   ← 花费 30 币，下次打工收益 +50%
```

- 合约只能预约一次
- 合约在打工完成时自动生效
- 如果打工期间被牛，合约作废（不退费）

#### c) 打工搭档（社交协作）

```
老婆 打工 搭档 @用户   ← 邀请对方一起打工
```

- 两人同时让老婆打工 → 双方都获得 +10% 收益
- 但打工期间如果任何一方被牛，搭档关系解除
- 每日限 1 次搭档

#### d) 打工排行榜

| 排行榜 | 维度 | 奖励（每周） |
|--------|------|------------|
| 打工收入榜 | 本周打工总收入 | 前 3 获得称号 |
| 打工连续榜 | 当前连续打工天数 | 前 3 获得单抽券 |

### 5.8 打工经济影响

**净收益计算**（以普通打工为例）：
```
启动费: -10 币
收益: +15~20 币
净赚: +5~10 币
风险: 被牛概率 ×1.5（约从 15% 升到 22.5%）
```

**博弈分析**：
- 休闲玩家：每天打工 1 次，净赚 5-10 币，30 天 +150~300 币
- 风险玩家：每天远征打工，净赚 20-30 币，但被牛概率高
- 大佬：老婆多，让闲置老婆打工，被牛了也不心疼

**与 NTR 的对冲**：
- 取消防连环牛 → 多人可以围攻同一大佬
- 打工让大佬老婆更容易被牛 → 动态平衡
- 大佬可以选择：不打工（安全但缺钱）vs 打工（赚钱但有风险）
- 这就是**正和博弈**：玩家有选择，群聊有动态

## 七、PK 系统升级

### 7.1 属性克制

```
力量型(atk最高) → 敏捷型(hp最高) → 智力型(def最高) → 力量型
```

- 克制方伤害 ×1.2，被克方 ×0.8（总倍率 1.5 倍）
- 玩家可以**选择派哪只老婆出战**

### 7.2 战力公式

```
power = (atk + def + hp*0.5) × (1 + intimacy_bonus) × (1 + bond_bonus)
  intimacy_bonus = intimacy / 500        # 100亲密 = +20%
  bond_bonus = 同作品数 × 0.05
  🆕 work_penalty = 0.15/0.25/0.35（打工期间）
```

### 7.3 战力公式数值验证

| 稀有度 | 基础战力 | 亲密0 | 亲密100 | 满羁绊 |
|--------|---------|-------|---------|-------|
| N | 40 | 40 | 48 | 51 |
| R | 70 | 70 | 84 | 89 |
| SR | 120 | 120 | 144 | 154 |
| SSR | 190 | 190 | 228 | 243 |

- SSR 满配 vs N 裸抽 = 6 倍差距 → 稀有度有意义
- SR 满配 vs SSR 裸抽 = 0.81 → 养成可追平 → ✅

### 7.4 奖励重构

| 结果 | 旧 | 新 |
|------|-----|-----|
| 胜方 | 15 币 + 图鉴 | 15 币 + 图鉴 + 赛季积分 +5 |
| 败方 | 0 | 5 币 + 赛季积分 +1 |
| 平局 | 随机胜负 | 双方各 8 币 |

## 十三、实施风险与回滚策略

### 13.1 风险评估

| 风险 | 概率 | 影响 | 缓解 |
|------|------|------|------|
| 打工系统被滥用刷币 | 中 | 中 | 打工收益有上限，被牛风险存在 |
| NTR + 打工双重打击导致退坑 | 低 | 高 | 新手保护 + 首次安抚 |
| 远征打工收益过高 | 低 | 中 | 被牛概率 ×2.5，清零风险高 |
| 经济通胀 | 中 | 中 | 打工是天然 sink（启动费 + 被牛风险） |

### 13.2 灰度策略

Phase 4A 先在 1-2 个活跃群试运行 3 天：
- 监控 NTR 次数变化
- 监控打工参与率
- 监控币经济平衡

### 13.3 回滚预案

所有新功能通过配置开关控制：
```yaml
work_enabled: false  # 关闭打工系统
intimacy_decay.enabled: false  # 关闭亲密度衰减
```
