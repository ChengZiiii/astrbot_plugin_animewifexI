# v3 离婚系统设计文档

> 玩家主动触发"离婚"——放弃某位老婆换取老婆币返还，但有概率被分走"家产"（正资产按比例收，负债也按比例减压）。
> 日期：2026-07-12
> 版本：v3.0 离婚系统 beta
> 关联文档：`game_design/v3_接力战_主线.md`（v3.1 深化版）·`game_design/玩法升级策划案.md`（v2 基线）

---

## [S1] 一句话描述

`老婆 离婚 <编号>` 触发：玩家主动放弃持有某位老婆，按"稀有度 + 好感度"返还老婆币，**有概率被分家产**（按当前余额 20% 双向比例，正分正/负分负；绝对值封顶 200）。

---

## [S2] 核心变化 vs 当前

| 维度 | 当前（Phase 4） | 升级后 |
|------|----------------|--------|
| 主动放弃老婆 | 无机制（只有 NTR 被抢） | **老婆 离婚** 命令 |
| 老婆币返还 | 无 | 稀有度基础 × (1 + 好感度/100) |
| 风险 | 无 | 概率分家产（双向比例） |
| 二次确认 | 无 | 必须显式确认 |
| 冷却 | 无 | 全用户级 7 天 |
| 图鉴 | 持有即在图鉴 | 离婚后保留图鉴、移除持有 |

## [S5] 返还公式（核心收益）

```python
def calc_divorce_return(rarity: str, intimacy: int) -> int:
    base = {"N": 10, "R": 25, "SR": 60, "SSR": 120}[rarity]
    mult = 1 + intimacy / 100.0          # 100 好感度 = ×2
    return int(base * mult)
```

| 老婆举例 | 基础 + 好感倍率 | 离婚返还 |
|---------|------------|---------|
| N + 好感 0 | 10 × 1.00 | 10 币 |
| R + 好感 50 | 25 × 1.50 | 37 币 |
| SR + 好感 80 | 60 × 1.80 | 108 币 |
| SSR + 好感 100 | 120 × 2.00 | **240 币** |

**经济直觉**：SSR+满好感 = 240 币 ≈ 8 次单抽。这是"养完卖"的天花板，**养 SSR + 离婚 = "无风险刷币"**——但有分家产概率制约。

## [S6] 分家产概率公式

```python
def calc_divorce_property_split_prob(intimacy: int) -> float:
    p = 0.10 + intimacy / 200.0
    return min(0.50, p)                   # 好感度 100 → 50% 概率封顶
```

| 好感度 | P(分家产) |
|--------|-----------|
| 0 | 10%（新手玩家很安全）|
| 20 | 20% |
| 50 | 35% |
| 80 | 50%（封顶）|
| 100 | 50%（封顶）|

**核心张力**：好感度越高 → 返还越多，但分家产概率也越大。等价于"用心养的老婆 = 卖时风险更高"。

## [S7] 分家产金额公式（Q4 双向百分比）

> 用户决策：现实中的离婚分家产是按百分比分，所以**正分正、负分负**——负债也算家产的一部分。

```python
def calc_divorce_property_split_amount(current_coins: int) -> int:
    """Q4 双向百分比 ±200 封顶。
    
    Returns:
        分走的金币量；正数 = 玩家损失金币，负数 = 玩家被减压（负债减少）
    
    Examples:
        余额 +500  → 被分走 +100  → 新余额 +400
        余额   0  → 被分走   0   → 新余额   0
        余额 -500  → 被分走 -100  → 新余额 -400（负债减压 20%）
        余额 +2000 → 被分走 +200  → 新余额 +1800（封顶）
        余额 -2000 → 被分走 -200  → 新余额 -1800（封顶）
    """
    split = int(current_coins * 0.20)
    if split > 200:
        return 200                        # 正向封顶
    if split < -200:
        return -200                       # 负向封顶
    return split
```

### [S7.1] 新余额计算

```python
def apply_divorce_split(current_coins: int) -> tuple[int, int]:
    """返回 (new_coins, split_amount)。"""
    split_amount = calc_divorce_property_split_amount(current_coins)
    return (current_coins - split_amount, split_amount)
```

| 当前余额 | 分走量 | 新余额 | 体感 |
|---------|--------|--------|------|
| +2000 | +200 | +1800 | 大户被扣 200（封顶）|
| +500 | +100 | +400 | 中等损失 |
| +100 | +20 | +80 | 小损失 |
| 0 | 0 | 0 | 无变化 |
| -100 | -20 | -80 | 负债减压 20 币 |
| -500 | -100 | -400 | 负债减压 100 币 |
| -2000 | -200 | -1800 | 巨负减压 200 币（封顶）|

## [S8] 二次确认 + 冷却

```python
# Cooldown（用户级，不是老婆级——离婚对玩家心理影响大）
DIVORCE_COOLDOWN_DAYS = 7
```

### 命令流程

```
老婆 离婚 <编号>
   ↓
检查：持有该老婆 ✓、不在战斗中 ✓、不在打工中 ✓
   ↓
检查：上次离婚日期距今 ≥ 7 天 ✓
   ↓
计算预览：
   "⚠️ 你确定离婚 [三笠 SSR ❤️90] 吗？
    返还：240 币
    分家产概率：50%
    分家产金额（如果当前余额 X）：±Y 币"
   ↓
老婆 离婚 <编号> 确认   ← 必须二次确认
   ↓
执行：
   1) 扣减余额（返还 + 已应用分家产）
   2) 删除 ownership（wid 离开持有列表）
   3) 写 Profile.last_divorce_date
   4) 累计 total_divorces / total_divorce_coins_earned / total_divorce_property_lost / total_divorce_debt_relieved
   5) 发结算贴
```

### 冷却规则

```
last_divorce_date 与今天日期差 < 7 天 → "你刚离过婚，7 天内不能再次离婚（你可以改主意，但不能反复）"
```

## [S10] 图鉴状态（Q2 决策）

```python
# 离婚后图鉴不变
# Profile.collection 保留 wid
# Ownership 删除（持有列表减少 1）
# 图鉴展示："[wid] 已离异"（轻提示）
```

**理由**：
- 保留图鉴 = 不惩罚也不奖励（"清晰"）
- 移除图鉴 = 强惩罚（玩家会愤怒："我好不容易抽到的忘了"）
- 标记图鉴 = 增加情绪负担（轻度负面体验）

>Q2 决策"保留"避免了极端两面性，最适合群聊小游戏定位。

## [S11] 边界与异常

### [S11.1] 战斗中老婆不能离婚
```python
if ownership.is_in_battle:  # Phase 4 已有的标记
    return "她正在战斗中，结束后再说"
```

### [S11.2] 打工中老婆不能离婚
```python
if ownership.is_working:
    return "她正在打工中，等结算后再说"
```

### [S11.3] 离婚后 + 重新抽到同一老婆
允许——历史 wid 重新触发，可能因为持有 wid 都从全局图鉴随机。不影响离婚系统。

### [S11.4] 离婚返还后余额仍为负（连分家产减压都没救回来）
- 例：离婚前余额 -1500，离婚返还 240，分家产减压 200（绝对值封顶），新余额 -1060
- 这是正常的：离婚返还 > 分家产减压时玩家净赚
- 但如果离婚返还 < 分家产减压绝对值，理论上新余额可能更低（比如离婚前 -1800、被分走 -200、离婚返还 +80，新余额 -1920）
- **接受这种情况**——离婚本质是"放弃老婆"，同时"结清部分家产"

### [S11.5] 锁定老婆能否离婚？
- **可以**：锁定不影响离婚触发，但离婚会清空锁定字段
- **理由**：锁定是"防 NTR 偷走"，跟"自己放弃"独立

### [S11.6] 老婆正在 PK 战斗中不能离婚
> 与 §S11.1 合并：所有"使用中"状态都不能离婚

## [S14] 经济影响测试（沙盒演算）

### 案例 1：休闲玩家（好感度 20，余额 +100）
- 返还 = 25 × 1.2 = 30（离婚 N 卡 R 好感 20）
- P(分家产) = 0.10 + 20*0.005 = 0.20
- 平均期望收入 = 30 + 0.20 × (-20) = **+26 币**
- **温和经济，玩家小赚**

### 案例 2：忙碌玩家（好感度 80，余额 +500，养 SSR）
- 返还 = 120 × 1.8 = 216
- P(分家产) = 0.50（封顶）
- 平均期望 = 216 + 0.50 × (-100) = **+166 币**
- **大赚但承担风险**

### 案例 3：负债玩家（好感度 100，余额 -2000，养 SSR）
- 返还 = 120 × 2.0 = 240
- P(分家产) = 0.50
- 平均期望 = 240 + 0.50 × (-(-200)) = 240 + 100 = **+340 币**
- **巨负减压神器**——这就是 Q4 双向百分比的对负债玩家的福利

### 案例 4：刷分玩家（一周内反复离婚 N 卡）
- 7 天冷却阻止——**第一次离婚后必须等 7 天**
- 这是 Q3 决策"7 天冷却"的价值

### 经济闭环验证

离婚系统的"产出"对负债系统的"减压"是**纯正贡献**——玩家多了一种"放弃 + 减压"的选择，而不是被强制刷打工。

## [S17] 已知局限与未来扩展

### 未来扩展候选（不阻塞首发）

- 离婚历史记录（翻看上次离婚明细）
- 离婚辩护 / 复婚机制
- 彩礼系统（结婚时付出彩礼币，离婚时返还按规则）
- 离婚冷静期倒计时（3 天后才真正执行）
- 离婚保险（消耗 coin 让分家产概率减半）

## [S18] 与负债系统的对接点

这是 v3_离婚.md 与接下来要写的 `v3_经济_负债_分币.md` 的边界：

| 离婚这边做的事 | 负债系统那边做的事 |
|--------------|------------------|
| 离婚返还币 → 余额 += 返还 | 余额 <= 0 时进入负债 |
| 分家产双向公式 → 余额按 20% 缩（带封顶）| 负债余额每天累计利息 |
| 离婚不深究"余额怎么变负" | 负债系统关心"为什么负" |

**关键约束**（离婚 → 负债 must not break）：

1. 离婚返还 `+` 分家产 都是 **int 整数运算**，不分正负币
2. 离婚不能导致余额 < `debt_floor_min`（防止离婚把玩家打到绝对底线以下，具体值由负债系统定义）
3. 离婚累计 `total_divorce_property_lost` 和 `total_divorce_debt_relieved` —— 这些统计指标可能被负债系统用作"是否享受负债减免"的依据

> **结论**：离婚系统的接口已经稳定，无论负债系统如何实现，"分家产双向比例"语义不变。
